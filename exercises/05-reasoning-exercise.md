# 05 | 练习：让 Agent 学会推理

## 🎯 练习目标

- 掌握 CoT（思维链）的提示词设计方法
- 理解 ToT（思维树）的多路径探索思路
- 能根据问题类型选择合适的推理策略

---

## 📋 练习要求

### 基础任务（必做）

**任务 1：设计一个 CoT 提示词**

选择一个复杂问题（如行程规划、数据分析、代码调试），设计一个 CoT 提示词。

要求：
1. 在 systemPrompt 中明确要求"分步思考"
2. 包含"因为...所以..."的推理展示
3. 用中文编写

**验证方式**：运行代码，观察 Agent 是否展示了清晰的推理步骤。

---

**任务 2：对比 CoT 和直接回答**

用同一个问题，分别用两种方式提问：

- **方式 A**：直接提问，不要求展示推理
- **方式 B**：加上 CoT 提示词要求分步思考

对比两次输出的质量差异。

**验证方式**：哪次的结果更可靠？为什么？

---

### 进阶任务（选做）

**任务 3：实现一个简化版 ToT**

用 Claude Agent SDK 实现 ToT 流程：

```
1. 生成问题："如何减少服务器的 API 响应时间？"
2. 生成 3 个不同方向的解决方案
3. 对每个方案进行评估（LLM 作为评审）
4. 根据评估选择最优方案
5. 对最优方案进行深入展开
```

**提示**：
- 可以用多个 `query()` 调用模拟 ToT 的不同阶段
- 每个阶段用不同的 systemPrompt 控制角色

---

## 🔍 思考题

1. CoT 能保证 LLM 的推理一定正确吗？如果推理步骤本身就是错的会怎样？
2. ToT 比 CoT 更"强大"，是不是所有问题都应该用 ToT？为什么？
3. 推理能力和记忆能力有什么关系？Agent 需要"记住"推理中间步骤吗？
