# 06 | 练习：让 Agent 学会自我纠正

## 🎯 练习目标

- 理解 ReAct 循环的原理和实现方式
- 能用外部验证（编译、测试）触发 Agent 自我纠正
- 掌握从"发现错误"到"修正错误"的完整闭环

---

## 📋 练习要求

### 基础任务（必做）

**任务 1：实现一个"编译 → 报错 → 修正"的自我纠正循环**

用 Claude Agent SDK 实现以下流程：

1. 让 Agent 生成一段"读取 JSON 文件并解析"的 TypeScript 代码
2. 故意引入一个错误（或者第一次就让它自己写）
3. 用 `tsc` 编译，捕获错误信息
4. 把错误信息反馈给 Agent，让它修正
5. 再次编译验证

**验证方式**：比较原始代码和修正后代码的区别。

---

**任务 2：ReAct 循环实现**

实现一个简化版的 ReAct 循环 Agent，用于解决以下任务：

**任务：随机想一个 1-100 之间的数字，然后让 Agent 猜**

Agent 每猜一次，你要告诉它"太大"或"太小"，它根据反馈调整，直到猜对。

**验证方式**：Agent 是否能在合理次数内猜对？它的推理过程合理吗？

---

### 进阶任务（选做）

**任务 3：多轮自我纠正直到测试通过**

实现一个循环：

```
循环直到单元测试全部通过（最多 10 轮）：
  1. Agent 生成一段代码
  2. 运行单元测试
  3. 如果有测试失败 → 把错误反馈给 Agent → Agent 修正 → 继续
  4. 如果全部通过 → 结束
```

观察：实际需要几轮才能让测试全部通过？Agent 是否在重复同样的错误？

---

## 🔍 思考题

1. 自我纠正是否可能陷入"无限循环"？如何设计终止条件？
2. 什么情况下应该让 Agent 自我纠正，什么情况下应该直接告诉它答案？
3. 自我纠正和"重新生成"有什么区别？哪个代价更大？
