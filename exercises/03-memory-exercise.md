# 03 | 练习：让 Agent 拥有记忆

## 🎯 练习目标

- 理解会话上下文的机制和局限
- 能用 SDK 实现多轮对话中的信息保持
- 能用向量数据库实现跨会话的长期记忆

---

## 📋 练习要求

### 基础任务（必做）

**任务 1：会话上下文——让 Agent 记住用户名字**

用 Claude Agent SDK 实现一个 5 轮对话的 Agent：

- 第 1 轮：用户自我介绍（名字 + 职业 + 所在城市）
- 第 2-5 轮：用户随意提问，Agent 需要在回答中自然使用第 1 轮的信息

**验证方式**：运行代码，观察 Agent 能否在第 3、4、5 轮中主动提及用户的名字/职业/城市。

**提示**：
- SDK 的 `query()` 默认不携带历史消息
- 可以在循环外维护一个 `messages` 数组，每次 `query()` 后把用户和 Agent 的消息都 push 进去
- 下次 `query()` 时，把这个数组作为参数传入

---

**任务 2：跨会话记忆——用 ChromaDB 存储和检索**

用 ChromaDB（内存模式）实现：

1. **存储记忆**：把下面 3 条信息存入向量数据库
   - "用户的生日是 1995 年 3 月 15 日"
   - "用户对 React 技术栈最熟悉"
   - "用户在上个月完成了项目 A"

2. **检索记忆**：用"用户最近做了什么项目？生日是什么时候？"查询

3. **注入回答**：把检索结果注入 systemPrompt，让 Agent 正确回答问题

**验证方式**：Agent 能否准确回答生日和项目信息。

---

### 进阶任务（选做）

**任务 3：分析 Context Window 截断行为**

写一个 Agent 对话，让它持续聊 30 轮（可以设计成让 Agent 每次回复"This is message X of 30"）。

观察：
- 第几轮开始 Agent 不再记得用户的名字（第 1 轮说的）？
- 这种现象和 token 限制有什么关系？

---

## 🔍 思考题

1. 如果用户的记忆信息和当前对话内容有冲突（比如用户以前说喜欢咖啡，现在说戒了咖啡），Agent 应该以哪个为准？
2. 持久化记忆的"Embedding 模型"如果换掉了（比如从中文模型换成英文模型），已经存储的记忆还能用吗？为什么？
3. 向量数据库检索和普通数据库的精确搜索，在 Agent 记忆这个场景下各有什么优缺点？
