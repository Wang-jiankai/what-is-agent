# 04 | 练习：让 Agent 学会反思

## 🎯 练习目标

- 理解"生成 → 反思 → 修正"循环的设计思路
- 能用双角色提示词实现结构化反思
- 能将 CRITIC 框架应用到实际评审任务中

---

## 📋 练习要求

### 基础任务（必做）

**任务 1：实现"生成 → 反思 → 修正"循环**

用 Claude Agent SDK 实现一个反思循环：

1. **任务**：让 Agent 写一个"判断字符串是否是回文"的函数
2. **第一轮**：生成初稿
3. **第二轮**：评审者检查（提示：检查边界情况如空字符串、单字符、包含空格/标点）
4. **第三轮**：根据评审意见修正

**验证方式**：比较初稿和修正版的区别，确认修正版处理了边界情况。

---

**任务 2：使用 CRITIC 框架评审一篇文章**

选择一个你熟悉的领域（如编程、烹饪、运动），让 Agent 写一段介绍（约 200 字）。

然后用 CRITIC 框架反思这段文字，最后输出一版改进版。

**验证方式**：对比原版和改进版，指出至少 2 处改进。

---

### 进阶任务（选做）

**任务 3：设计一个多轮迭代直到"评审通过"**

实现一个循环：

```
循环直到评审者说"通过"（最多 5 轮）：
  - 执行者生成内容
  - 评审者检查
  - 如果有问题 → 修正 → 继续循环
  - 如果没问题 → 输出最终版本
```

观察：实际需要几轮才能达到"通过"标准？

---

## 🔍 思考题

1. 反思是否总是必要的？什么情况下反思反而是多余的？
2. 如果让 Agent 自己生成评审标准（而不是我们给定 CRITIC），会有什么区别？
3. 反思和测试的关系是什么？能否用自动化的测试替代人类的反思？
