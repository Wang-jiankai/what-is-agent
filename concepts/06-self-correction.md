# 06 | 自我纠正（Self-Correction）

## 🎯 学习目标

- 理解什么是 Agent 的自我纠正能力
- 掌握 ReAct 循环的核心原理（推理→行动→观察→调整）
- 能设计"试错→发现→修正"的 Agent 循环
- 理解自我纠正与反思的区别

---

## 📖 概念讲解

### 什么是自我纠正？

自我纠正 = 发现自己错了 + 主动改正。

人类在执行复杂任务时，不是第一次就能做对：

```
尝试 → 发现错误 → 调整策略 → 再次尝试 → 直到成功
```

**自我纠正（Self-Correction）** 是 Agent 具备的相同能力——不是等着用户说"你错了"，而是主动发现错误并修正。

### 自我纠正 vs 反思（区别）

| 概念 | 回答的问题 | 本质 |
|------|-----------|------|
| **反思**（Reflection） | "我做对了吗？有什么问题？" | 检查 |
| **自我纠正**（Self-Correction） | "发现问题了，怎么修？" | 修正 |

**关系**：反思发现问题 → 自我纠正修复问题 → 再次反思验证修复

## 上篇：ReAct 循环

**ReAct = Reasoning + Acting + Observing**

ReAct 是最经典的自我纠正框架，核心是一个循环：

```
Loop:
  ┌─────────────────────────────────┐
  │ 1. Think（推理）                 │
  │    "我当前的状态是什么？         │
  │     下一步应该做什么？"
  └──────────────┬──────────────────┘
                 ▼
  ┌─────────────────────────────────┐
  │ 2. Act（行动）                   │
  │    执行工具调用或生成输出
  └──────────────┬──────────────────┘
                 ▼
  ┌─────────────────────────────────┐
  │ 3. Observe（观察）               │
  │    收集行动的结果                 │
  │    "这次行动产生了什么？"
  └──────────────┬──────────────────┘
                 ▼
  ┌─────────────────────────────────┐
  │ 4. Adjust（调整）               │
  │    基于观察判断：                 │
  │    - 成功了？→ 继续或结束        │
  │    - 失败了？→ 回到推理，重新规划│
  └──────────────┬──────────────────┘
                 │
                 └── 循环直到任务完成
```

### ReAct 的关键思想

ReAct 的核心洞察：**推理和行动不是分离的两个阶段，而是交织在一起。**

每次行动后，Agent 不是盲目继续，而是"边做边想"，根据结果动态调整。

### ReAct 示例：调试一个 Bug

```
问题：用户反映登录功能无法使用

Loop 1:
  Think：我需要先了解问题的具体情况。让我检查日志。
  Act：读取日志文件
  Observe：日志显示 "token expired" 错误
  Adjust：错误是 token 过期导致的，需要刷新 token

Loop 2:
  Think：应该修改 token 刷新逻辑。
  Act：修改 token 刷新代码
  Observe：代码修改成功
  Adjust：还需要测试

Loop 3:
  Think：让我测试一下修复是否有效。
  Act：运行登录测试
  Observe：登录成功 ✓
  Adjust：问题已解决，任务完成
```

## 下篇：在 Agent 中实现自我纠正

### 方法一：通过 systemPrompt 注入自我纠正意识

```
你是一个谨慎的助手。在完成任务时：

1. 每执行一步，问自己"结果对不对？"
2. 如果发现错误，不要重复同样的做法
3. 思考"换个方法能解决这个问题吗？"
4. 不断尝试直到成功，或者明确报告无法解决
```

### 方法二：外部验证触发纠正

```
用户输入 → Agent 执行 → 验证器检查结果
                              ↓
                        结果正确？
                        ↓         ↓
                       是        否
                        ↓         ↓
                      输出    触发纠正循环
```

外部验证器可以是：
- 编译器/类型检查（代码错误）
- 单元测试（功能正确性）
- 规则引擎（逻辑一致性）
- 人类反馈

### 方法三：多版本生成 + 选择

```
1. 生成多个版本的解法/回答
2. 对每个版本进行评审
3. 选择最好的一个，或综合各版本的优点
```

## 串联：从反思到自我纠正

完整的 Agent 认知循环：

```
         ┌──────────────────────────────────┐
         │           Agent 认知循环           │
         └──────────────────────────────────┘

   推理 ──→ 行动 ──→ 观察 ──→ 反思 ──→ 纠正
    ↑                              │
    │                              │
    └──────────────────────────────┘
                  继续循环
```

| 阶段 | 核心活动 | 关键问题 |
|------|---------|---------|
| **推理** | 制定计划 | "我应该怎么做？" |
| **行动** | 执行计划 | "我现在执行这一步" |
| **观察** | 收集反馈 | "产生了什么结果？" |
| **反思** | 检查结果 | "我做对了吗？" |
| **纠正** | 调整策略 | "怎么修正错误？" |

---

## 💡 生活中的类比

**学骑自行车的过程：**

```
第一次骑：上车 → 摇摇晃晃 → 摔倒
  Think: 我为什么会摔倒？
  Observe: 车把太偏右了
  Adjust: 下次把车把放正

第二次骑：上车 → 还是偏 → 这次反应快，修正了方向
  ...
第 N 次：终于能平稳骑行了
```

每次摔倒都是一个"Observe"，然后思考为什么摔倒，下次调整——这就是自我纠正。

---

## 📝 章节回顾

**记住这三个关键点：**

1. **自我纠正是主动的，不是被动的**
   不是等用户说"你错了"，而是 Agent 自己发现错误并改正

2. **ReAct 是自我纠正的经典框架**
   推理→行动→观察→调整→循环，直到任务完成

3. **反思 vs 纠正 = 检查 vs 修复**
   反思发现"错了"，纠正解决"怎么改"
