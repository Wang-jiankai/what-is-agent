# 02 | 规划（Planning）

## 🎯 学习目标

- 理解为什么 Agent 需要"规划"能力
- 掌握任务分解的基本方法
- 了解两种常见的规划模式

---

## 📖 概念讲解

### 什么是规划？

**规划（Planning）** 是 Agent 将复杂任务分解为多个可执行的子步骤，并按合理顺序执行的能力。

想象你要完成一道菜："做一盘番茄炒蛋"。你会自然地分解为：

```
1. 准备食材（番茄、鸡蛋、葱）
2. 洗菜
3. 打蛋
4. 切番茄
5. 热锅
6. 炒蛋
7. 炒番茄
8. 混合翻炒
9. 装盘
```

这就是人类的"规划能力"。Agent 也需要同样的能力。

### 为什么 Agent 需要规划？

| 场景 | 无规划的 Agent | 有规划的 Agent |
|------|-------------|--------------|
| 用户说："帮我搭建一个博客" | 可能直接乱写代码 | 先分析需求，列出步骤，逐步执行 |
| 遇到错误 | 可能重复犯错 | 先分析错误原因，调整计划 |
| 复杂任务 | 容易迷失方向 | 按计划有条不紊推进 |

### 任务分解的基本方法

#### 方法一：线性分解

适合**有明确顺序**的任务。

```
目标：泡一杯咖啡
分解：
  1. 把水加入水箱
  2. 放入咖啡粉
  3. 按下启动按钮
  4. 等待萃取完成
```

#### 方法二：并行分解

适合**可同时执行**的子任务。

```
目标：准备早餐
分解（可并行）：
  - 任务 A：烤面包（3分钟）
  - 任务 B：煮咖啡（5分钟）
  - 任务 C：煎蛋（2分钟）

同时开始，总耗时 = max(3,5,2) = 5分钟
```

### 两种常见的规划模式

#### 1. Plan-and-Execute（计划然后执行）

```
Step 1: 制定完整计划
        "这个任务需要 A、B、C、D 四个步骤"
Step 2: 按顺序执行
        A → B → C → D

特点：先想清楚再动手，适合长任务
```

#### 2. ReAct（推理+行动混合）

```
Loop:
  Step 1: Think（推理）  "我下一步应该做什么？"
  Step 2: Act（行动）    "执行这个工具"
  Step 3: Observe（观察） "查看结果"
  Step 4: 判断是否继续

特点：边做边想，适合探索性任务
```

## 💡 systemPrompt 与结构化指令

规划模式是通过 `systemPrompt` 来实现的。具体做法是把"先计划再执行"或"边做边想"的要求写在 `systemPrompt` 里。

`systemPrompt` 决定了 Agent 的行为方式。把行为规范写成清晰的结构化指令，就是一种"**结构化提示词**的实践。

如果你想深入学习如何写高质量的结构化指令（把专家经验固化成 SOP），请参考：

## 💡 生活中的类比

想象你要**组织一次旅行**：

**Plan-and-Execute 风格（计划先行）：**
> 先查攻略→定行程→订机票酒店→打包行李→出发。全程按计划走。

**ReAct 风格（边做边调整）：**
> 先出发，订个去程机票→到了当地再根据天气决定去哪儿→临时订酒店→遇到感兴趣的地方多待两天。

两种风格没有绝对好坏，根据任务性质选择。

## 📝 章节回顾

**记住这两个关键点：**

1. **规划的本质是任务分解**
   把复杂目标拆成简单步骤，按顺序或并行执行

2. **两种规划模式**
   - Plan-and-Execute：先计划再执行，适合稳定、长线任务
   - ReAct：边做边想，适合动态、探索性任务
