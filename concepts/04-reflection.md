# 04 | 反思（Reflection）

## 🎯 学习目标

- 理解什么是 Agent 的"反思"能力
- 掌握让 Agent 检查自己输出的方法
- 了解反思与自我纠正的关系
- 能设计"生成 → 反思 → 修正"的循环

---

## 📖 概念讲解

### 什么是反思？

人类在完成任务后，会回头检查：

> "做得怎么样？有没有问题？下次怎么改进？"

**反思（Reflection）** 是 Agent 具备的类似能力——在执行后回头审视自己的输出，判断是否正确、是否完整、是否有改进空间。

### 为什么 Agent 需要反思？

LLM 的输出是**概率生成**，不是完美推理。即使模型很强，也可能：
- 遗漏用户问题的部分要求
- 生成看似合理但实际错误的代码
- 在复杂推理中跳步

反思机制让 Agent 有机会**主动发现这些问题**，而不是直接输出一个可能有缺陷的结果。

### 反思 vs 自我纠正（区别）

| 概念 | 回答的问题 | 阶段 |
|------|-----------|------|
| **反思**（Reflection） | "我做对了吗？有什么问题？" | 执行后，检查阶段 |
| **自我纠正**（Self-Correction） | "发现问题了，怎么修？" | 检查后，修正阶段 |

**关系**：反思 → 发现问题 → 自我纠正 → 再次反思 → ...

---

## 上篇：让 Agent 检查自己

### 核心方法：双角色提示

最简单的反思实现——让 Agent 用**两个身份**：

```
身份 1（执行者）：完成任务，输出结果
身份 2（评审者）：审查执行者的输出，指出问题
```

### 代码示例：带反思的代码生成

```
用户：请写一个排序函数

↓ 第一步：执行者生成代码
function bubbleSort(arr) { ... }

↓ 第二步：评审者反思
"这段代码正确吗？时间复杂度是多少？有没有边界情况没处理？"

↓ 第三步：执行者根据评审意见修正
function bubbleSort(arr) {
  // 增加了边界检查
  // 优化了时间复杂度
}
```

### 什么情况下反思最有用？

| 场景 | 反思的价值 |
|------|-----------|
| 代码生成 | 发现逻辑漏洞、边界问题 |
| 复杂推理 | 发现跳步或前提错误 |
| 长文本生成 | 检查是否遗漏要点、逻辑是否连贯 |
| 数据分析 | 验证结论是否有数据支撑 |

## 下篇：结构化反思框架

### CRITIC 反思框架

一个实用的结构化反思方法：

| 字母 | 含义 | 自问 |
|------|------|------|
| **C** | Correctness（正确性） | 我的输出在事实和逻辑上正确吗？ |
| **R** | Relevance（相关性） | 我回应了用户真正的问题吗？ |
| **I** | Completeness（完整性） | 我遗漏了什么吗？ |
| **T** | Truthfulness（真实性） | 我在编造信息吗？ |
| **I** | Insight（洞察） | 我能提供更深层的分析吗？ |
| **C** | Coherence（连贯性） | 我的输出逻辑通顺吗？ |

### 代码示例：CRITIC 反思循环

```
用户：请介绍一下 React 的虚拟 DOM

↓ 执行者输出
"虚拟 DOM 是 React 的核心特性..."

↓ CRITIC 反思
- Correctness ✓ 虚拟 DOM 基本概念正确
- Relevance ✓ 回答了问题
- Completeness ⚠️ 没提 fiber 架构
- Truthfulness ✓ 没有编造
- Insight ⚠️ 没解释为什么虚拟 DOM 性能好
- Coherence ✓ 结构清晰

↓ 修正后的输出
（在原来基础上增加了 fiber 架构和性能分析）
```

## 串联：反思在 Agent 循环中的位置

一个完整的 Agent 循环：

```
     ┌──────────────┐
     │   感知环境    │  用户输入、工具返回
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │   规划步骤    │  决定下一步做什么
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │   执行行动    │  调用工具、生成输出
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │   反思检查    │  ← 这一章的重点
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │   自我纠正    │  发现问题就修正
     └──────┬───────┘
            ▼
         继续？
```

## 💡 生活中的类比

**写作的反思过程：**

初稿 → 检查 → 发现问题 → 修改 → 再检查 → 定稿

Agent 的反思就像写作中的"检查"步骤——初稿不一定完美，但通过反思可以不断提升质量。

---

## 📝 章节回顾

**记住这三个关键点：**

1. **反思是主动检查，不是被动等待**
   反思让 Agent 在输出前主动发现问题，而不是依赖用户说"你错了"

2. **反思 ≠ 纠正**
   反思回答"对不对"，纠正回答"怎么改"

3. **结构化反思更可靠**
   CRITIC 等框架让反思不遗漏重要维度
