# 07 | 工具调用（Tool Use）

## 🎯 学习目标

- 理解什么是 Agent 的"工具调用"能力
- 掌握 Claude Agent SDK 的工具注册和调用方式
- 理解工具的定义、执行和结果处理
- 了解如何设计自定义工具

---

## 📖 概念讲解

### 什么是工具调用？

**工具（Tool）** 是 Agent 用来与外部世界交互的接口。

没有工具的 Agent 只能"思考"——有工具的 Agent 能够"行动"。

| 能力 | Agent 能做什么 |
|------|---------------|
| 仅 LLM | 接收文本，输出文本（思考） |
| + 工具 | 执行代码、读写文件、搜索网页（行动） |

### Claude Agent SDK 的工具调用

Claude Agent SDK 通过 `allowedTools` 配置允许 Agent 使用哪些工具：

```typescript
for await (const message of query({
  prompt: "帮我读取当前目录下的 package.json",
  options: {
    // 允许 Agent 使用这些内置工具
    allowedTools: ["Bash", "Read", "Write", "Edit", "Glob"],
  },
})) {
  console.log(message);
}
```

### 内置工具一览

| 工具 | 能力 | 使用场景 |
|------|------|---------|
| **Bash** | 执行 shell 命令 | 运行脚本、安装依赖 |
| **Read** | 读取文件内容 | 查看代码、配置文件 |
| **Write** | 写入文件 | 创建/覆盖文件 |
| **Edit** | 修改文件部分内容 | 修复 bug、调整代码 |
| **Glob** | 文件模式匹配 | 查找文件、批量操作 |
| **Grep** | 文本内容搜索 | 在代码中查找关键字 |
| **WebSearch** | 搜索网页 | 获取最新信息 |

## 上篇：工具的定义与注册

### 工具是什么？

在 Claude Agent SDK 中，工具本质上是一个**函数**：

```
工具 = 函数名 + 输入参数 + 功能描述 + 返回值
```

SDK 通过 `allowedTools` 告诉 Agent："你有这些工具可以用，你需要的时候调用它们。"

### 工具调用流程

```
用户输入 → Agent 分析 → 发现需要工具 → 调用工具 → 获取结果 → 继续推理 → 输出
```

### 什么是 MCP 工具？

MCP（Model Context Protocol）是一种**工具的标准协议**。

如果把工具比作"USB 设备"，MCP 就是"USB-C 接口"——一种统一的工具连接标准。

> 更多关于 MCP 的内容，请参考：[what-is-mcp 仓库](https://github.com/Wang-jiankai/what-is-mcp)

## 下篇：设计自定义工具

### 自定义工具的三个步骤

```
1. 定义：写出工具的名称、描述、参数
2. 注册：把工具加入 Agent 的可用列表
3. 执行：Agent 根据需要调用
```

### Claude Agent SDK 的工具限制

**重要**：Claude Agent SDK（v0.2.x）的 `allowedTools` 只接受**内置工具名称字符串**：

```typescript
// ✓ 支持：字符串形式的内置工具
allowedTools: ["Bash", "Read", "Write"]

// ✗ 不支持：自定义函数直接传入
allowedTools: [myCustomTool]  // ❌ 这样不行
```

### 如何扩展自定义工具？

SDK 本身不直接支持自定义函数工具，但有以下方案：

| 方案 | 说明 |
|------|------|
| **Bash 包装** | 用 Bash 调用任意脚本 |
| **MCP 协议** | 通过 MCP 服务器接入自定义工具 |
| **Tool Use 循环** | 自己实现一个 ReAct 循环，手动处理工具调用 |

## 💡 生活中的类比

**工具 = 主厨的厨房设备：**

| 工具 | 厨房对应 |
|------|---------|
| Bash | 烤箱、炉灶 |
| Read/Write | 冰箱（存取食材） |
| WebSearch | 市场采购（获取外部食材） |

主厨不是什么都用手，能用设备的用设备，才能高效完成任务。

## 📝 章节回顾

**记住这三个关键点：**

1. **工具让 Agent 从"思考"到"行动"**
   没有工具，Agent 只能输出文字；有了工具，Agent 能操控世界

2. **Claude Agent SDK 通过 allowedTools 控制可用工具**
   目前只支持内置工具名称（字符串），不支持直接传自定义函数

3. **MCP 是工具的标准化协议**
   如果需要自定义工具，通过 MCP 协议接入是官方推荐方式
