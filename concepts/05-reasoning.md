# 05 | 推理（Reasoning）

## 🎯 学习目标

- 理解什么是推理能力，以及为什么 LLM 本身具备推理能力
- 掌握两种核心推理技术：CoT（思维链）和 ToT（思维树）
- 了解 Claude SDK 的 thinking 功能（beta）
- 能设计"推理 → 行动"的 Agent 循环

---

## 📖 概念讲解

### 什么是推理？

**推理（Reasoning）** 是从已知信息出发，通过逻辑步骤得出新结论的过程。

LLM 本身就是一个**推理引擎**——当你给它一个数学题或者逻辑题，它在生成每个词的时候，实际上在做推理。

### 为什么 Agent 需要专门的推理能力？

基础 LLM 的推理是"隐式"的——它生成答案，但**不展示推理过程**。这带来两个问题：

| 问题 | 表现 |
|------|------|
| **不可审计** | 你不知道它怎么得出的结论 |
| **推理深度有限** | 复杂问题可能跳步，导致错误 |

专门的推理能力让 Agent 能够：
- **显式展示思考步骤**——像人类一样"先想再答"
- **深入多步推理**——不只一步到位，而是多步验证
- **探索多条路径**——一个问题尝试多种解法，再选最优

---

## 上篇：CoT — 思维链（Chain of Thought）

### 核心思想

不要直接给答案，让 LLM **先写出推理过程**。

**没有 CoT：**
```
用户：3 × (4 + 5) = ?
LLM：3 × (4 + 5) = 27
```

**有 CoT：**
```
用户：3 × (4 + 5) = ?
LLM：先算括号里的：4 + 5 = 9
     再乘法：3 × 9 = 27
     答案是 27
```

### CoT 的关键要素

有效的 CoT 提示词应该：

1. **明确要求分步**："请一步一步思考"
2. **中间步骤可见**：让 LLM 说出"因为...所以..."
3. **逻辑连接词**：首先、其次、因此、然而

### 代码示例：CoT 提示词

```
请解决以下问题。对于每一步，请写出你的推理过程。

问题：小明有 10 个苹果，给了小红 3 个，又买了 5 个，现在有多少个？

推理过程：
第一步：小明原来有 10 个苹果
第二步：给了小红 3 个，10 - 3 = 7
第三步：又买了 5 个，7 + 5 = 12
最终答案：12 个
```

### CoT 适用场景

| 适合 | 不适合 |
|------|--------|
| 数学计算 | 事实查询（"首都是什么"） |
| 逻辑推理 | 创意写作 |
| 代码调试 | 简单分类 |

## 下篇：ToT — 思维树（Tree of Thought）

### 核心思想

CoT 是**线性**推理——一条路走到底。ToT 是**树状**探索——同时尝试多条路径，选最优。

```
问题：如何优化这个数据库查询？

          ┌─ 方案 A：加索引
         ╱
根问题 ───┼─ 方案 B：重写 SQL
         ╲
          └─ 方案 C：增加缓存

评估每条路径 → 剪枝不好的分支 → 深入好的分支 → 最终答案
```

### ToT 的四个步骤

| 步骤 | 动作 |
|------|------|
| **Thought**（思考） | 生成一个可能的想法 |
| **Evaluate**（评估） | 判断这个想法的好坏 |
| **Prune**（剪枝） | 丢弃明显不好的分支 |
| **Expand**（扩展） | 继续深入好的分支 |

### ToT 适用场景

| 场景 | ToT 的价值 |
|------|-----------|
| 复杂规划 | 探索多种方案，评估风险 |
| 代码优化 | 尝试多种优化策略 |
| 战略决策 | 权衡多个选项的利弊 |

## 串联：推理在 Agent 循环中的位置

```
     ┌──────────────┐
     │   感知环境    │
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │   推理思考    │  ← CoT/ToT 在这里
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │   执行行动    │
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │   反思检查    │
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │   自我纠正    │
     └──────┬───────┘
```

推理（Reasoning）是 Agent **制定计划**时的大脑活动；反思（Reflection）是 Agent **检查输出**时的评审活动。

## 💡 生活中的类比

**走路去一个陌生的地方：**

**CoT（线性思考）：**
> "先往北走两个路口，看到超市右转，再走五分钟就到了。"

**ToT（树状探索）：**
> "有三条路可以走——路线 A 经过高速但有收费，路线 B 最近但可能堵车，路线 C 绕远但风景好。我选 B，先出发。走着发现堵车，再切到 C。"

---

## 📝 章节回顾

**记住这三个关键点：**

1. **CoT 让推理可见**
   "一步一步思考"不只是提示技巧，它让 LLM 的推理过程透明化

2. **ToT 让推理可探索**
   复杂问题不只有一条路，ToT 让 Agent 能够权衡多条路径

3. **推理 ≠ 行动**
   推理是思考阶段，行动是执行阶段。两者结合才能形成完整的 Agent 循环
