# 02 | 参考答案：让 Agent 学会规划

## 📋 基础任务参考答案

**为"泡一杯绿茶"设计规划步骤**

### 线性分解（完整步骤）

```
1. 把水加入水壶
2. 把水壶放到加热底座上
3. 打开电源，加热水至沸腾（约100°C）
4. 等待水温度降至80-85°C（绿茶适宜温度）
5. 取出茶叶罐和茶杯
6. 向茶杯中放入适量茶叶（约3-5克）
7. 将热水倒入茶杯
8. 等待1-2分钟浸泡
9. 取出茶漏或过滤茶叶渣
10. 享用绿茶
```

### 可并行的步骤分析

以下步骤可以**同时进行**：

| 并行组 | 步骤 | 说明 |
|--------|------|------|
| 组 A | 步骤 2-4 | 烧水的同时可以准备其他东西 |
| 组 B | 步骤 5、6 | 取出茶叶罐和茶杯可以与烧水同时进行 |

**更优化的并行计划：**

```
T=0:  开始烧水 + 取出茶叶罐和茶杯（并行）
T=1:  水烧开，等待降温
T=2:  放入茶叶
T=3:  倒入热水
T=4:  等待浸泡
T=5:  过滤茶叶渣，开始享用
```

## 📋 进阶任务参考答案

| 场景 | 推荐模式 | 理由 |
|------|---------|------|
| 1. 预订一周旅行行程 | **Plan-and-Execute** | 目标明确、步骤固定（订机票→订酒店→安排景点），提前规划效率更高 |
| 2. 在陌生城市找餐厅 | **ReAct** | 信息不完整，需要边搜索边试探，根据实时反馈调整 |
| 3. 写 5000 字技术博客 | **Plan-and-Execute** | 任务稳定，先定大纲再填充内容比边写边改更有条理 |

## 🔍 思考题答案

**1. 依赖关系需要注意什么？**

> 规划时必须**严格按依赖顺序**排列步骤。比如"先注册才能登录"，注册必须在登录之前。可以用有向图来表示依赖关系。

**2. 为什么有些任务"计划赶不上变化"？**

> 因为任务执行过程中会遇到**不可预知的反馈**——比如搜索结果、API 返回值、外部环境变化等。这类任务适合 **ReAct**，边执行边调整计划。

**3. Agent 规划 vs 人类规划，区别在哪？**

> 本质区别在于：
> - **人类**：依赖经验、直觉、对世界的常识理解
> - **Agent**：依赖 LLM 的推理能力和给定的工具/API
> 共同点是：都需要把大目标分解成小步骤，都需要根据结果调整计划。
